# 2026-07-02 작업 일지

## 개요

Claude Code PTY 래퍼(wrap)에서 승인 프롬프트(approval prompt)를 감지하고, 데몬을 통해 모바일 클라이언트로 전달하는 **승인 알림 파이프라인** 구축.

---

## 변경 파일 (3개, +167 / -11)

### 1. `internal/detector/event.go` — 툴 승인 필터링

- `needsApproval(toolName)` 함수 추가
  - 사용자 승인이 필요한 툴만 필터링: `Bash`, `Write`, `Edit`, `NotebookEdit`, `AskUserQuestion`, `Skill`, `WebFetch`
  - Read-only / 자동 승인 툴은 무시하고 조용히 스킵
- `Feed()` 메서드에서 기존 3초 타이머 로직을 `needsApproval()` 조건으로 변경

### 2. `internal/server/ws.go` — `/approval` HTTP 엔드포인트

- `handleApproval()` 핸들러 추가
  - wrap PTY stdout 스캐너로부터 `POST /approval` 요청 수신
  - JSON 바디에서 `type`, `description`, `toolName` 파싱
  - `Hub.HandleAlert()`를 통해 WebSocket으로 모든 모바일 클라이언트에 승인 프롬프트 전파
- `Start()`에서 `/approval` 라우트 등록

### 3. `internal/wrap/wrap.go` — 승인 프롬프트 감지 로직 (주요 변경)

#### 구조 변경
- `Command()`: 기존 `io.Copy(os.Stdout, tty)` → `scanAndRelay(tty, os.Stdout, 9171)` 로 교체
- 단순 PTY→stdout 릴레이에서 **패턴 매칭 기반 승인 감지**로 업그레이드

#### 새 함수들

| 함수 | 역할 |
|------|------|
| `scanAndRelay()` | PTY 출력을 읽어 stdout에 쓰면서 승인 프롬프트 감지. 최근 2KB tail 버퍼 유지. |
| `isApprovalPrompt()` | 4가지 패턴으로 Claude 승인 요청 감지 |
| `lastQuestionLine()` | tail 버퍼에서 마지막 질문 라인 추출 |
| `notifyDaemon()` | 감지된 승인 프롬프트를 `POST /approval` 로 데몬에 전송 |

#### 감지 패턴 (4가지)

1. **전통적 (y/n)**: `(y/n)` 또는 `(y/N)` 포함
2. **번호 선택 메뉴**: `"1. Yes"` + `"No"` 동시 포함 (Claude Code 기본 UI)
3. **물음표 종결**: 라인이 `?`로 끝남
4. **의도 질문**: `"do you want"` 또는 `"proceed?"` 포함 (대소문자 무시)

#### 중복 방지

- `alertedFor` 변수로 동일 라인 재알림 방지

#### 침묵 감지 (백업 메커니즘)

- 마지막 출력이 물음표로 끝난 후 2초간 새 출력이 없으면 승인 프롬프트로 재판단
- goroutine으로 비동기 실행, tail 버퍼 비교로 새 출력 도착 여부 확인

---

## 데이터 흐름

```
Claude Code PTY 출력
  → wrap.scanAndRelay() 패턴 매칭
    → wrap.notifyDaemon() HTTP POST
      → daemon /approval 핸들러
        → WebSocket → 모바일 클라이언트 (승인 알림)
```
