# 06. 경쟁 환경 분석 (Competitive Landscape)

AI 코딩 에이전트 통제 시장은 아직 절대 강자가 없는 초기 시장(Blue Ocean)입니다.

## 1. 기존 우회망(Tailscale + RDP/SSH)과의 정면 비교
현재 개발자들이 Aider나 Claude Code를 폰에서 제어하기 위해 사용하는 방식은 다음과 같습니다:
* **경쟁 방식:** `VPN(Tailscale) 켜기` → `모바일 SSH 앱 켜기` → `tmux 세션 접속`
* **우리의 우위 (Phase 1→2):**
  * **속도:** 앱 하나만 켜면 백그라운드 데몬과 1초 만에 WebRTC로 직통 연결됩니다.
  * **UX:** 폰의 좁은 화면에서 SSH로 타이핑하는 고통을 [DevKeypad]와 [안전 Paste] 버튼이 완벽히 해소합니다. Diff 텍스트를 드래그하거나 코드 덩어리를 밀어 넣는 행위 자체가 혁신적으로 편해집니다.

## 2. 모바일 전용 AI 대시보드(스타트업들)와의 비교
* **경쟁 방식:** CodeAgent Mobile 등 API 기반 대시보드. 에이전트 로그를 예쁘게 파싱해 보여줌.
* **우리의 우위:** API 파싱 앱은 에이전트가 예상치 못한 에러(예: Git 충돌, 환경 변수 누락)로 터졌을 때 폰에서 할 수 있는 게 없습니다. 우리는 **PTY(가상 터미널) 직결 및 실제 화면 스트리밍**을 지원하므로, 에이전트가 막히면 개발자가 직접 터미널에 개입해 문제를 풀고 작업을 이어갈 수 있는 '하이브리드' 유연성을 제공합니다.

## 3. 솔직한 약점(Weakness) 인정 및 완화 전략
* **약점:** WebRTC 영상 스트리밍은 텍스트(SSH)나 JSON API 뷰어보다 네트워크 대역폭과 모바일 배터리 소모가 심합니다.
* **완화(Mitigation):** 
  * 평상시에는 PTY 기반의 텍스트 로그만 DataChannel로 가볍게 주고받으며 화면 렌더링을 일시 중지합니다.
  * 복잡한 Diff 리뷰나 IDE 환경 확인이 필요할 때만 선택적으로 비디오 스트리밍을 켜는 '토글(Toggle)' 방식을 도입하여 배터리 효율과 네트워크 대역폭 한계를 극복합니다.
