# 03. 인프라스트럭처 비용 (Infrastructure Cost)

본 프로젝트의 비즈니스 핵심은 중앙 서버 유지비용을 극한으로 압축하여 **"이익률 90% 이상의 마이크로 SaaS"** 구조를 만드는 것입니다.

## 1. Firebase 시그널링 유지비 (Cost: $0 지향)
* **기능:** PC IP 교환(1KB 미만 트래픽) 및 기기 페어링 확인.
* **보안/최적화:** 
  * Firebase Security Rules를 엄격하게 적용하여, 인증된 사용자(Firebase Auth)가 자신의 노드(`users/{uid}/devices`)에만 접근하도록 제한합니다.
  * E2EE(종단간 암호화)된 IP 문자열 하나만 저장하므로, DB 용량 비용은 사실상 발생하지 않습니다. 무료 티어(Spark Plan)로 MAU 수만 명까지 커버 가능합니다.

## 2. P2P 릴레이(TURN/Relay) 서버 비용 (Cost: 최소화)
* **현실적 가정:** 사용자의 약 10~15%는 통신사 방화벽 문제로 직통 연결(P2P)이 실패합니다. 이들을 위해 우리가 트래픽 중계를 해줘야 합니다.
* **비용 절감 기술:** 
  * WebRTC는 기본적으로 영상 압축(H.264/VP9)이 뛰어나 대역폭 소모가 적습니다.
  * 릴레이 서버 트래픽이 월 1TB를 초과할 경우, 해당 헤비 유저에게만 "릴레이 무제한 패스" 명목으로 월정액 과금($5/mo)을 유도하여 릴레이 인프라 적자를 원천 방어합니다.

## 3. 숨겨진 비용: 고객 지원 (Customer Support) 부담
* 네트워크 연결 불량(이중 NAT, 방화벽 차단 등)은 사용자마다 환경이 달라 CS 문의가 빗발칠 수 있는 최대 리스크입니다.
* **대응 전략:** 앱 내 네트워크 진단 툴을 내장하여, 연결 실패 시 "직통 연결 실패! 릴레이 모드로 전환합니다." 혹은 "사내 보안망 차단으로 연결할 수 없습니다"라는 명확한 진단 메시지를 띄워 불필요한 CS 티켓 발행을 억제합니다.
