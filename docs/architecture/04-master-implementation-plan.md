# 04. 마스터 구현 계획 (Master Implementation Plan)

이 문서는 AI 관제 모바일 커맨드 센터, DevRemote의 장기 기술 로드맵입니다.

## Phase 1: 프로토타입 및 UX 검증 (완료)
* **목표:** RustDesk 포크를 활용하여 "모바일에서 코딩 화면을 보고 통제하는 폼팩터" 자체의 시장성 검증.
* **구현 내역:**
  * 기존 하단 툴바 전면 철거 및 코딩 전용 플로팅 버튼(FAB) 통합.
  * 모바일 키보드 스트레스 원천 차단: 클립보드 동기화 기반 **[안전 Paste 800ms]** 기능 구현.
  * 뷰포트 강제 고정용 **[Zoom Lock]** 탑재.
  * 초기 터치 모드 강제 진입 적용.

## Phase 2: WebRTC 완전 자체 개발 & PTY 데몬 (현재 준비 중)
* **목표:** AGPL-3.0 라이선스에서 벗어나 독자적인 코어 엔진 구축.
* **세부 태스크:**
  1. **Firebase Auth & 시그널링 구축:** 구글 로그인 등 Firebase 인증을 붙이고, 보안 룰을 세팅하여 1:1 기기 IP 교환소 마련.
  2. **E2EE / mTLS 도입:** 교환되는 IP를 암호화하고 상호 인증서 생성.
  3. **자체 PTY 생성기 (Terminal Daemon):** 
     * 데스크톱 화면 전체를 스트리밍하는 것이 무거울 경우, 혹은 백그라운드에서 AI 에이전트를 조작해야 할 경우, 기존 화면을 '가로채는(Hijacking)' 대신 데몬 자체가 가상 터미널(PTY) 세션을 생성합니다.
     * 이 PTY의 stdout/stderr 텍스트 스트림만 WebRTC DataChannel로 모바일에 가볍게 쏴주어, 비디오 스트리밍 없이 텍스트 기반의 완벽한 CLI 원격 제어도 구현 가능케 합니다.
  4. **화면 캡처 (ScreenCaptureKit):** 모바일에서 Diff 창이나 브라우저 확인이 필요할 때만 선택적으로 WebRTC 비디오 트랙 활성화.

## Phase 3: 상용화 및 오픈 코어 (미래)
* 오픈소스 공개 및 커뮤니티 빌딩.
* 결제 모듈(Stripe) 연동 및 엔터프라이즈 팀 관리 기능 구축.
