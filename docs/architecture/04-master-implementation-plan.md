# 04. 마스터 구현 계획 (Master Implementation Plan)

이 문서는 AI 관제 모바일 커맨드 센터, DevRemote의 장기 기술 로드맵입니다.

## Phase 1: 프로토타입 및 UX 검증 (완료)
* **목표:** RustDesk 포크를 활용하여 "모바일에서 코딩 화면을 보고 통제하는 폼팩터" 자체의 시장성 검증.
* **구현 내역:**
  * 기존 하단 툴바 전면 철거 및 코딩 전용 플로팅 버튼(FAB) 통합.
  * 모바일 키보드 스트레스 원천 차단: 클립보드 동기화 기반 **[안전 Paste 800ms]** 기능 구현.
  * 뷰포트 강제 고정용 **[Zoom Lock]** 탑재.
  * 초기 터치 모드 강제 진입 적용.

## Phase 2: 투트랙(Two-Track) 아키텍처 및 컴패니언 데몬 (현재 준비 중)
* **목표:** 수십 주의 개발 지연(Reinventing the wheel)을 방지하기 위해 RustDesk 코어는 유지하되, 비즈니스 핵심인 'AI 제어 모듈'만 독립 프로세스로 분리하는 하이브리드 아키텍처 채택.
* **아키텍처 피봇의 배경 (왜 Clean Room을 포기했는가?):**
  * OS 레벨의 화면 캡처(ScreenCaptureKit, DXGI)와 하드웨어 가속 인코딩, 권한 탈취 로직을 WebRTC와 밑바닥부터 결합하는 것은 최소 12~20주가 소요되는 하드코어 엔지니어링임.
  * RustDesk가 5년간 해결해 둔 크로스 플랫폼 엣지 케이스들을 굳이 버리고 데스밸리에 빠질 이유가 없음.
* **세부 태스크 (Two-Track 분리):**
  1. **Track 1 (Raw 제어망 - RustDesk 유지):** 화면 스트리밍, 키보드/마우스 주입, NAT 홀펀칭 등 무거운 OS 레벨 기능은 기존에 빌드해 둔 RustDesk 포크를 그대로 활용함.
  2. **Track 2 (AI 브레인 - 컴패니언 데몬 신규 개발):** 
     * RustDesk와는 완전히 분리된 독립 프로세스(Process)로 컴패니언 데몬을 생성.
     * 이 데몬은 터미널 텍스트(PTY)를 가로채고 AI 에이전트의 상태를 파싱하는 비즈니스 로직 전담.
     * MIT 라이선스의 가벼운 라이브러리(`pion/webrtc` 또는 `webrtc-rs`)를 사용하여 모바일의 DevKeypad와 1:1 데이터 채널(DataChannel) 통신 수행.
  3. **Firebase Auth & 시그널링 구축:** 컴패니언 데몬과 모바일 간의 IP 교환 및 보안 인증(mTLS) 체계 구축.

## Phase 3: 상용화 및 오픈 코어 (미래)
* 오픈소스 공개 및 커뮤니티 빌딩.
* 결제 모듈(Stripe) 연동 및 엔터프라이즈 팀 관리 기능 구축.
