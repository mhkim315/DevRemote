# 01. 프로젝트 개요 (Project Overview)

## 1. 프로젝트 비전
**"AI 코딩 에이전트를 침대 위에서, 이동 중에도 완벽하게 통제한다."**
DevRemote는 Devin, Aider, Claude Code 등 자율형 AI 코딩 에이전트들이 쏟아지는 시대에, 개발자가 데스크톱 앞을 떠나서도 에이전트의 상태를 직관적으로 모니터링하고 파괴적 작업(Destructive Action)을 안전하게 승인(Approve)할 수 있게 해주는 **모바일 특화 씬 클라이언트(Thin Client)**입니다.

## 2. 해결하고자 하는 핵심 문제 (Pain Points)
* **데스크 의존성:** AI 에이전트가 코드를 분석하고 작성하는 수십 분의 시간 동안 사용자는 PC 앞을 떠날 수 없음.
* **승인 병목 (HITL):** 파일 삭제, DB 수정 등 중요한 결정의 순간에 에이전트가 사용자의 Y/N 대답을 기다리며 멈춰 있음.
* **기존 RDP의 모바일 한계:** AnyDesk나 TeamViewer는 범용 목적이라 모바일 화면에서 스크롤, 줌, 텍스트 드래그(복사)가 극도로 불편하며, 특히 긴 코드를 폰에서 붙여넣을 때 `\n` 때문에 강제 엔터가 쳐지는 치명적 결함이 있음.

## 3. 핵심 솔루션 (하이브리드 접근법)
우리는 기존 스타트업들의 '단순 로그 뷰어' 방식을 거부하고, **'직접 화면 스트리밍 + 코딩 특화 키패드 주입'**이라는 하이브리드 방식으로 접근합니다.
* 터미널 뷰포트를 화면에 고정하는 **[Zoom Lock]**
* 한 번에 안전하게 프롬프트를 밀어 넣는 **[안전 Paste]**
* 에이전트의 런타임 상태를 가로채는 **[컴패니언 데몬(Daemon)]**

## 4. 3중 보안 아키텍처 체계
우리의 앱은 사용자의 개발 머신(루트 권한)에 접근하므로 극강의 보안이 필요합니다.
* **1단계 (시그널링 방어):** Firebase를 통해 PC와 모바일의 IP를 교환할 때, 서로만 아는 비밀키로 E2EE(종단간 암호화) 처리하여 중앙 서버나 해커가 IP 자체를 볼 수 없게 만듭니다.
* **2단계 (인증 방어):** mTLS (Mutual TLS) 상호 인증을 구현하여, 클라이언트와 데몬 양쪽이 서로의 인증서(Certificate)를 확인하지 않으면 연결 자체를 거부합니다.
* **3단계 (권한 방어):** 모바일에서 실행할 수 있는 명령어(Command)를 데몬 수준에서 '에이전트 제어'에 필요한 것으로만 화이트리스트(Whitelist) 처리하여 랜섬웨어 감염 시나리오를 원천 차단합니다.

## 5. 오픈소스 해자(Moat) 전략
경쟁사가 우리의 UI나 아키텍처를 카피하는 것을 막고 시장을 빠르게 장악하기 위한 무기입니다.
* 코어 기술(데몬 및 통신 로직)과 기본 모바일 앱을 오픈소스로 풀어버려 전 세계 개발자들이 자발적으로 개선하고 사용하게 만듭니다.
* 경쟁사가 진입하더라도 이미 커뮤니티가 구축된 '무료이자 가장 안전한' 오픈소스 표준을 이길 수 없습니다.
* 우리는 이 오픈소스 파이를 바탕으로, 팀 단위 관리 기능이나 클라우드 릴레이 서버 사용권 등을 '엔터프라이즈 플랜(Open Core)'으로 수익화합니다.
